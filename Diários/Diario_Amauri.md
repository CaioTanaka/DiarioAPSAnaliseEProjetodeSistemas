## Aula 3 - 04/03
Vimos sobre levantamento de requisitos, entendi que é a fase de coletar as informações necessárias para saber oque o sistema deve fazer, nessa aula a atividade foi pesquisar sobre as técnicas de levantamento, que são: entrevista que fica mais detalhada se as perguntas forem bem feitas, questionário que é rápido e em escala parecido com a entrevista mas com perguntas mais objetivas, observação ver oque acontece de verdade, documentos que é visualizar regras já existentes, JAD que são trocas de ideias em grupo, protótipo que é testar ideia visualmente, etnografia onde o pesquisador é introduzido no ambiente de trabalho do sistema, workshop acontece um debate mediado para discutir sobre oque o sistema deve ter, brainstorming é uma reunião de pessoas que vão dando suas ideias de acordo com o tema.

## Aula 8 - 08/04
Aqui foi onde foi apresentado sobre histórias de usuários, cenários e prototipação e oque é cada etapa dessa. O objetivo agora era organizar com oque já tínhamos, e ver se realmente entendemos o sistema e transformar em um protótipo de telas mas antes disso fazer as historias de usuários que são quem faz, oque acontece, qual regra importa e qual resultado de acontecer, e também para cada história os cenários dela. Dividimos para o grupo cada um fazer um módulo tipo cadastro, empréstimo, devolução, etc... tudo isso com base em tudo que já tínhamos. Minha parte foi o modulo de cadastro então para realizar entendi quem que cadastrava o usuário que no caso é a bibliotecária, oque cada tipo de usuário tem de diferente do outro (professor, aluno e funcionário) com base no que já tínhamos, sempre bom lembrar.
